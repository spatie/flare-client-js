# v1.2.0 Payload Format Migration — Design

**Status:** Approved
**Date:** 2026-04-29
**Scope:** Final v1 release of `@flareapp/js`, `@flareapp/react`, `@flareapp/vue`. Ships the Flare v2 wire format from a v1 public API.

## Goal

Publish `@flareapp/js@1.2.0`, `@flareapp/react@1.2.0`, `@flareapp/vue@1.2.0` such that:

1. Public API of all three packages is byte-identical to their last released versions (`@flareapp/js@1.1.0`, `@flareapp/react@1.0.1`, `@flareapp/vue@1.0.1`).
2. Egress is the v2 wire format: new endpoint, new headers, OTel-style attribute payload, glows-as-span-events. Server's legacy `MapOldReportFormatAction` is no longer in the path.
3. Four security/data-loss bug fixes from PR #39 ride along.
4. v1.2.0 is the **final** v1 release. Subsequent work goes to v2.

Reference for the canonical v2 wire shape: PR #39 (`task/update-payload-format`).

## Non-goals

- No new public API. Specifically: no `setEntryPoint`, `setSdkInfo`, `setFramework`, no vue router option, no new react attribute exports.
- No deprecation warnings on solution providers. They keep working publicly; their output is silently dropped at the egress mapper.
- No release of the unreleased work currently sitting on `main` (FlareErrorBoundary, flareReactErrorHandler, vue TS migration, tsdown migration). That stays on `main` for a future release.
- No backport of fileReader caching, vue idempotent install, resetKeys diff fix, light-debug, seenAt-at-entry, or any other PR #39 fix not in the four below.

## Branch strategy

`main` has unreleased v2-bound work that is incompatible with a v1 baseline. The v1.2.0 release must be cut from the last clean v1 state.

```
e1fa9b3  Release 1.1.0                 ← @flareapp/js@1.1.0 tag
…
dafeac5  Release flareapp/react 1.0.1  ← @flareapp/react@1.0.1 tag
─────────────────────────────────────── BASE for the long-lived 1.x branch
```

1. Create long-lived branch `1.x` from commit `dafeac5c2d49511023fb3583c185d13d1ec264d8`. Push to remote. This branch parks all v1 code.
2. Create feature branch `feature/v1.2.0-payload-format` from `1.x`.
3. PR targets `1.x`.
4. After merge, tag `@flareapp/js@1.2.0`, `@flareapp/react@1.2.0`, `@flareapp/vue@1.2.0` on `1.x`.
5. `main` is untouched. Future v1 patch work, if needed, branches off `1.x`.

**Build chain check before starting work:** `dafeac5` predates the tsdown migration. Verify `npm run build`, `npm run test`, `npm run typescript` succeed on `1.x` as-is. If they don't, cherry-pick the infra-only commits (`6d208cc` deps update, `952886f` tsdown migration, `f30edcd` package.json cleanup, `6538802` typescript fixes) into `1.x` before opening the feature branch — never the feature commits (`fde5560`, `e3da5bf`, `64cde0b`, `705d2e9`, `2b7bb0a`, `ca96c37`, `2e013bf`, `7664264`, `df6b4e4`, `b9095b7`, `fd303b5`, `9827987`, `ddc6132`, `bdafb52`, `bc7f241`, `d5cc60a`, etc.).

## Architecture

```
flare.report(error)
  → Flare.buildReport()       [unchanged — emits v1 Report]
  → beforeEvaluate hook         [unchanged]
  → beforeSubmit hook           [unchanged — receives v1 Report]
  → Api.report(v1Report, config)
       └─ mapToV2Wire(v1Report, config)   [NEW — pure fn]
       └─ fetch(reportingUrl, {headers: v2Headers, body: V2WirePayload})
```

Mapper sits at the egress boundary. Everything above the mapper sees v1. Everything from the mapper down is v2. `beforeSubmit` receives the v1 `Report` — its public contract is preserved.

### New file: `packages/js/src/api/mapToV2Wire.ts`

Pure function. No side effects. No I/O. Fully unit-testable.

```ts
export function mapToV2Wire(report: Report, config: Config): V2WirePayload
```

### New file: `packages/js/src/api/v2WireTypes.ts`

Internal types only. Not re-exported from `packages/js/src/index.ts`. Consumers must not import them.

- `V2WirePayload`
- `V2StackFrame`
- `V2SpanEvent`
- `V2Attributes`

### Modified: `packages/js/src/api/Api.ts`

- Body becomes `flatJsonStringify(mapToV2Wire(report, config))` instead of `{...report, key}`.
- Headers: drop `X-Requested-With`, drop `X-Report-Browser-Extension-Errors`. Add `Accept: application/json` and `X-Flare-Client-Version: 2`. Keep `X-Api-Token`. Keep `Content-Type: application/json`.
- Success status check changes from `204` to `201`.
- Internal signature changes: `Api.report(report, config)` instead of `(report, url, key, reportBrowserExtensionErrors)`. The mapper needs `config.version`, `config.stage`, `config.sourcemapVersion`; the fetch needs `config.reportingUrl` and `config.key`. Internal-only — not part of the public API.

### Modified: `packages/js/src/Flare.ts`

- Default `reportingUrl`: `'https://reporting.flareapp.io/api/reports'` → `'https://ingress.flareapp.io/v1/errors'`.
- `Api.report()` call site updated to pass config.
- Solution providers still execute (`getSolutions(...)` runs as today). The resolved `Solution[]` populates `report.solutions` on the v1 Report. The mapper drops it. Trade-off: cycles spent resolving solutions go to waste, kept to keep the public surface bit-identical.

## Mapping table — v1 Report → v2 wire

### Top-level

| v1 source | v2 wire | Notes |
|---|---|---|
| `report.exception_class` | `exceptionClass` | string |
| `report.message` | `message` | string |
| `report.seen_at` (seconds) | `seenAtUnixNano` | `Math.round(seen_at * 1e9)` |
| `report.sourcemap_version_id` | `sourcemapVersionId` | top-level, not under attributes |
| `report.stacktrace` | `stacktrace` | per-frame mapping below |
| `report.glows` | `events` | via `glowsToEvents` below |
| `report.solutions` | **discarded** | no destination field in v2 |
| `report.notifier` | discarded as field | surfaced as attributes (below) |
| `report.language` | discarded as field | surfaced as attributes (below) |
| `report.stage` | discarded as field | surfaced as attribute `service.stage` |

### Stack frame (per-frame)

| v1 | v2 |
|---|---|
| `line_number` | `lineNumber` |
| `column_number` | `columnNumber` |
| `method` | `method` |
| `file` | `file` |
| `code_snippet` | `codeSnippet` (same shape) |
| `class` | `class` |
| `trimmed_column_number` | **discarded** |
| (none) | `isApplicationFrame` — heuristic: `false` if `file` matches `/node_modules|vendor|chunk-/`, else `true` |

### Glows → SpanEvents

```ts
{
  type: 'php_glow',
  startTimeUnixNano: Math.round(glow.microtime * 1e9),
  endTimeUnixNano: null,
  attributes: {
    'glow.name': String(glow.name),
    'glow.level': glow.message_level,
    'glow.context': glow.meta_data ?? {},
  }
}
```

`type: 'php_glow'` is correct — the server's `SpanEventType` enum has no `js_glow` case (per PR #39 commentary).

### Attributes

Built fresh per report. Keys absent if source is null/undefined.

| Source | Attribute key |
|---|---|
| literal `'@flareapp/js'` | `telemetry.sdk.name` |
| `CLIENT_VERSION` | `telemetry.sdk.version` |
| literal `'javascript'` | `telemetry.sdk.language` |
| literal `'javascript'` | `flare.language.name` |
| literal `'web'` | `flare.entry_point.type` |
| `window.location.href` (when `typeof window !== 'undefined'`) | `flare.entry_point.value` |
| `config.stage` (when truthy) | `service.stage` |
| `config.version` | `service.version` |
| `report.context.request.url` | `url.full` |
| `report.context.request.useragent` | `user_agent.original` |
| `report.context.request.referrer` | `http.request.referrer` |
| `report.context.request.readyState` | `document.ready_state` |
| `report.context.request_data.queryString` | `url.query` (object) |
| `report.context.cookies` | `http.request.cookies` (object) |
| Custom context — see rule below | `context.custom` (nested object) |

#### Custom context rule

Two distinct paths land on `report.context`:

1. `flare.addContext(name, value)` populates `this.context.context[name]`. After the report-build merge `{...context, ...this.context}`, this lives at `report.context.context.<name>`.
2. `flare.report(error, {vue: ...})` (passed-context arg) lives at `report.context.<key>` — sibling to `request`, `cookies`, etc.

Mapper rule:
- `report.context.context.<*>` → flattened so each child becomes a sibling under `attributes['context.custom'].<*>`.
- `report.context.<key>` for any `key` not in `{request, request_data, cookies, context}` → packed under `attributes['context.custom'].<key>`.
- Collision: addContext's bucket wins (matches existing spread precedence in `Flare.report`).

Rationale: this matches the v2 PR's `context.custom.react` / `context.custom.vue` convention, and keeps user-set custom context together in one nested bucket without namespace clashes against future server-reserved attribute keys.

## Bug fix backports

Each fix is its own commit. All four lands before mapper work so mapper tests run against corrected code.

### 1. cookie — preserve `=` in base64 values

**File:** `packages/js/src/context/cookie.ts`
**Bug:** `cookie.split(/=/)` truncates at first `=` — base64-encoded cookie values get cut off.
**Fix:** split-once via `indexOf('=')`. Cookies without `=` are skipped (no name= form is malformed).

### 2. catchWindowErrors — listener detach + non-Error rejections

**File:** `packages/js/src/browser/catchWindowErrors.ts`
**Bug A:** Property assignment `window.onerror = ...` is overwritten by any user code that also does `window.onerror = ...` after Flare initializes. Flare's handler silently disappears.
**Bug B:** `unhandledrejection` handler gates on `error.reason instanceof Error` and drops everything else (string rejections, plain object rejections).
**Fix A:** Use `window.addEventListener('error', ...)` and `window.addEventListener('unhandledrejection', ...)`. Drops the existing `originalOnerrorHandler` chaining (addEventListener doesn't displace prior listeners).
**Fix B:** Drop the `instanceof Error` gate. Coerce non-Error reasons to `new Error(typeof reason === 'string' ? reason : JSON.stringify(reason))` before calling `flare.report()`.

### 3. flatJsonStringify — broken decycle

**File:** `packages/js/src/util/flatJsonStringify.ts`
**Bug:** On a cycle hit, the replacer calls `JSON.parse(JSON.stringify(value))` which re-runs the same replacer on the same object graph and can blow up time/memory on shared sub-objects (legitimately appearing twice without cycle).
**Fix:** Replace with a `WeakSet`-based decycle. Seen objects are replaced by the literal string `'[Circular]'`. Linear time, no recursion.

### 4. createStackTrace — uncaught parser crash

**File:** `packages/js/src/stacktrace/createStackTrace.ts`
**Bug:** `ErrorStackParser.parse(error)` can throw on malformed stacks. Throw escapes the Promise constructor — report drops silently.
**Fix:** `try/catch` around `ErrorStackParser.parse`. On catch, resolve a single fallback frame: `file: 'unknown'`, `line_number: 0`, `column_number: 0`, `method: 'unknown'`, `code_snippet: {0: 'Could not parse stacktrace'}`, `trimmed_column_number: null`, `class: 'unknown'`.

## Test strategy

### Existing tests stay green

`packages/js/tests/{configure,context,glows,hooks,light,report,solutions}.test.ts` exercise public API behavior on the v1 `Report` shape. Mapper sits below `beforeSubmit`, so these tests do not need updating.

### `FakeApi` decision

`packages/js/tests/helpers/FakeApi.ts` continues to capture the v1 `Report` (mapper not invoked when FakeApi replaces real Api). Existing tests stay focused on v1 contract behavior.

### New tests

#### `packages/js/tests/mapToV2Wire.test.ts`

- Round-trip a fully populated v1 `Report` → assert exact v2 wire shape via golden fixture (`tests/fixtures/golden-v1-to-v2.json`).
- `seen_at` seconds → `seenAtUnixNano` rounding.
- `glows[]` → `events[]` with `php_glow` type, `microtime` → nanoseconds.
- `context.context.{foo, bar}` (addContext path) → `attributes['context.custom'].foo/.bar` flattened.
- `context.vue` (passed-context path) → `attributes['context.custom'].vue` packed.
- Collision: `addContext('vue', X)` + passed `{vue: Y}` → `attributes['context.custom'].vue === X`.
- `isApplicationFrame` heuristic: `node_modules` / `vendor` / `chunk-` path → false; app path → true.
- Solutions present in input → absent in output (no `solutions` key, no `flare.solutions` attribute).
- `sourcemap_version_id` → top-level `sourcemapVersionId`.
- Empty/missing optional fields don't emit `undefined` keys (mapper omits, doesn't include).

#### `packages/js/tests/Api.test.ts`

- `Api.report()` POSTs to `config.reportingUrl`.
- Headers include `X-Api-Token`, `X-Flare-Client-Version: 2`, `Accept: application/json`, `Content-Type: application/json`. Headers do **not** include `X-Requested-With` or `X-Report-Browser-Extension-Errors`.
- Body equals `flatJsonStringify(mapToV2Wire(report, config))`.
- Response status `201` is silent. Other statuses log `console.error`.
- Use `vi.spyOn(globalThis, 'fetch')` mock.

#### Bug fix tests

- `packages/js/tests/cookie.test.ts` — base64-style value preserved across `=` boundary.
- `packages/js/tests/catchWindowErrors.test.ts` — listener still fires after user `window.onerror = …` reassignment; non-Error rejection reason produces a report.
- `packages/js/tests/flatJsonStringify.test.ts` — direct cycle replaced with `'[Circular]'`; shared sub-object referenced twice doesn't blow up.
- `packages/js/tests/createStackTrace.test.ts` — error with malformed stack still resolves to one frame (the fallback).

### Endpoint flip — consumer impact

Default endpoint flips from `reporting.flareapp.io/api/reports` (port 443, legacy) to `ingress.flareapp.io/v1/errors` (port 443, new). Consumers with allowlist firewall rules on outbound HTTP must add the new host. CHANGELOG calls this out explicitly.
